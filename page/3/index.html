mysql的表锁和行锁

表锁(table lock)

表锁是mysql中最基本的所策略, 开销最小。会锁定整张表, 一个用户对表进行写操作前需要先获取写锁, 这会阻塞其它用户对该表的读写操作。在没有写锁的情况下, 用户可以获取读锁, 读锁之间是不互斥的, 也就是说多个用户可以同时加读锁, 但是不能加写锁。总结下来也就是写写互斥, 写读互斥, 读写互斥, 读读共享。因此读锁也称共享锁, 写锁称为排它锁。

Object clone()方法

打开Object类, 我们可以看到clone方法是一个本地方法。也就是说该方法是由java虚拟机实现的。
但是可以看到该方法的注释,该方法应该满足以下几点。
1) x.clone()!=x 原对象不等于克隆对象,相当于两个引用
2) x.clone().getClass() == x.getClass() 克隆对象与原对象的类型是一致的。
3) x.clone().equals(x) == true
