Thread常用API

发表于 2017-04-25 |

进程与线程

进程是受操作系统管理的基本运行单元,线程可以理解为进程中独立运行的一个子任务。
进程中至少包含一个线程。

java实现多线程的两种方式

  1. 继承Thread类,覆写run方法
  2. 实现Runable接口,覆写run方法,将该Runnable子类传给Thread构造方法中
    区别: 因为java只支持单继承的特点,一般实现线程选择实现Runable接口。

start()方法

start()方法用于启动线程,它会调用run方法,必须使用start方法才算启动线程。也就是说调用run
方法只是执行了run方法的内容,但是并没有启动线程。

Thread.currentThread()

获取当前线程

getName()

获取线程的名字

isAlive()

判断线程是否处于活动状态,只要线程已经启动并且尚未终止就是处于活动状态。

sleep(long millis)

使线程进入睡眠状态,如果在进入睡眠之前或者是在睡眠中别的线程中断了该线程则抛出
InterruptedException。此异常必须捕获。也就是说interrupt()会使线程从睡眠状态进入到
运行状态中。也就进入到catch代码块中。

yield()

使线程放弃cpu资源,当放弃的时间不确定。也有可能刚刚放弃执行权马上又抢到cpu资源进入到运行状态。

setPriority(int priority)/getPriority()

线程的优先级默认为5 依次从1-10。
并且线程的优先级具有继承性。也就是说A线程启动B线程,B线程的优先级与A一样。因为在创建线程时
即new Thread()将线程的优先级初始化了该线程的符线程的优先级。

setDaemon()

将线程设置为守护线程。
在java线程中有两种线程用户线程和守护线程。当进程中不存在非守护线程时守护线程会自动销毁。
最典型的线程就是垃圾回收线程。

interrupt()/isInterrupted()/interrupted()

  1. interrupt()改变线程的中断标志,使线程处于中断状态。
  2. isInterrupted()判断线程是否处于中断状态
  3. interrupted() 判断当前线程是否处于中断状态,所以此方法是静态方法

synchronized关键字

synchronized关键字是用来解决多个线程之间访问共享变量时出现非线程安全的问题

  1. 同步代码块 使用的锁是关键字所用的对象
  2. 同步函数 使用的是this 即当前对象
  3. 同步静态函数 使用的是该类的字节码对象

volatile关键字

只能用来修饰类的成员变量,主要作用使得它修饰的变量在多个线程之间是可见的。也就是说强迫每个线程从内存中重新读取该变量的值。注意该关键字不是原子性的操作,要解决安全问题还是得用synchronized

object类中的wait()/wait(long timeout)/notify/notifyAll()

这些方法只能用在同步代码中,也就是说必须由锁对象来调用这些方法。否则抛出不合法监视状态异常
IllegalMonitorStateException。
wait()使当前线程处于等待状态,并且释放锁资源。
wait(long timeout)当时间到了自动唤醒
notify()唤醒一个wait()中的一个线程
notifyAll唤醒所有wait状态的线程。
在调用wait时:如果线程处于中断状态或者处于wait状态时别的线程中断了该线程会抛出中断异常

join()/join(long millis)

方法join()的作用是使调用join方法的线程A正常执行run方法中的任务,而使当前线程B进行无线阻塞直至A线程运行完毕。
因为join内部使用了wait所以释放锁。

释放锁的时机

  1. 调用了wait()方法
  2. 出现了异常
  3. 调用了join方法

抛InterruptedException
